# Dotbot to Chezmoi Migration Implementation Plan

> **For Hermes:** Execute this plan task-by-task. This migration is intentionally non-destructive until an explicit `chezmoi apply` step.

**Goal:** Migrate Phil's existing Dotbot-managed main and local dotfiles into a simple, elegant, powerful chezmoi source that works across multiple macOS and Linux systems.

**Architecture:** Use `~/.local/share/chezmoi` as the single source of truth. Convert portable Dotbot symlinks into chezmoi files/templates, convert local/private config into `private_` files or encrypted/template data, and keep generated state out via `.chezmoiignore`. Validate each wave with `chezmoi diff` before applying.

**Tech Stack:** chezmoi v2.70.3, zsh/bash, XDG Base Directory spec, Homebrew/Brewfile, Git includeIf, 1Password/age for secrets, macOS `defaults`/LaunchAgents for host-specific setup.

**Objective:** Manage Homebrew environment/configuration while keeping `homebrew-brewfile` as the separate Brewfile collection repository.

- Create: `dot_config/homebrew/brew.env.tmpl`

- Create: `dot_config/shell/plugins/homebrew-file.sh`
- Do not import: `~/.config/homebrew/pjbeyer_homebrew-brewfile/`

**Steps:**

```bash

chezmoi execute-template < dot_config/homebrew/brew.env.tmpl >/tmp/brew.env && bash -n /tmp/brew.env
```

**Follow-up guidance:**

- Keep package lists, host Brewfiles, and domain Brewfiles in the external `homebrew-brewfile` repository.
- Use chezmoi for the environment variables, shell plugin wiring, and optional future bootstrap scripts that clone/update the external repo.
- Do not run `brew bundle` automatically until the constitution/specs define package-management safety gates.
